2. src/layers/RestrictedAirspaceLayer.ts:
   - On mount: loop through RESTRICTED_AIRSPACE, create a Cesium.Entity for each
   - Each entity uses PolylineGraphics (not PolygonGraphics) for the OUTLINE — closed loop
   - Material: Cesium.PolylineDashMaterialProperty with the zone's color, dashLength 16, gapLength 8
   - Width: 3 pixels
   - clampToGround: true
   - Optional: add a faint translucent fill using PolygonGraphics with same color at 0.05 alpha — this gives subtle visual presence at zoom-out without overwhelming
   - Click handler: store.setSelectedEntity({ type: 'airspace', id, data: zoneDefinition })
   - Subscribe to layerVisibility.restrictedAirspace — toggle entire collection
   - Add 'restrictedAirspace' to the store's layerVisibility object, default TRUE (these are core OSINT context, not optional)

3. LayerToggles.tsx — add a new toggle row "Restricted Airspace" between Jamming and Fires, red dot. Count = number of zones.

4. EntityPanel.tsx — handle type 'airspace': show name, description, an "i" icon. No coordinates display, no MMSI, no altitude — just the human-readable advisory.

2. src/utils/theaters.ts — function flyToTheater(viewer, theater):
   - viewer.camera.flyTo({
       destination: Cesium.Cartesian3.fromDegrees(lon, lat, height),
       orientation: { heading: 0, pitch: Cesium.Math.toRadians(pitch), roll: 0 },
       duration: theater.flyDuration,
       easingFunction: Cesium.EasingFunction.CUBIC_IN_OUT
     })
   - For each layer in theater.layers, store.setLayerVisible(layerName, value)
