P10.5 polish — satellite click-to-fly arrival is landing close but the satellite drifts out of frame on user zoom-in. Three adjustments:

1. Adjust satellite flyTo offset (params table in click-to-fly.ts, satellite row only):
   - Range: 50000 → 80000 m
   - Pitch: Cesium.Math.toRadians(-45) → Cesium.Math.toRadians(-20)
   - Heading: 0 (unchanged)
   Aircraft, vessel, fire offsets unchanged.

2. Predictive position for satellite click-to-fly:
   - flyTo duration is 1.5s. Satellites move ~7 km/s. Without prediction, the sat has moved 10+ km by the time camera arrives.
   - At click time, propagate satellite position 1.5s forward using satellite.js SGP4 on the main thread (not the worker — worker IPC adds latency we can't afford for a click handler). Import satellite.js directly into click-to-fly.ts if not already, use the satrec stored in the satellites Map for that NORAD ID.
   - Compute target time: Cesium.JulianDate.addSeconds(Cesium.JulianDate.now(), 1.5, new Cesium.JulianDate()).
   - Convert SGP4 ECI output to Cesium.Cartesian3 the same way the live propagation does (reuse existing helper if one exists, or inline transform).
   - If satrec lookup or propagation throws, fall back to currentPositions[satIndex] (current position). Log [SAT FLY PREDICT FAIL] norad=X reason=Y.
   - Apply ONLY to satellites. Aircraft/vessels/fires keep current-position targeting.

3. Rapid re-click handling:
   - If user clicks satellite B while a satellite-A click-to-fly is mid-flight: cancel A's flyTo (Cesium flyTo returns a Promise; track the active fly's cancel handle in click-to-fly.ts module state), destroy A's temp entity, then start B's predictive fly fresh.
   - Same applies cross-type: clicking a vessel mid-satellite-fly cancels the satellite fly.
   - Theater flythroughs still take priority — if isTheaterFlying is true, no click-to-fly executes regardless.

4. EntityPanel still opens immediately at click time with the satellite at its current (not predicted) position. Only the flyTo target uses predicted position.

Definition of done:
- Click satellite → camera flies to 80km off where the satellite WILL be in 1.5s, ~20° looking down, satellite centered in frame on arrival
- User can scroll-zoom toward satellite without it drifting off-screen
- glTF model visible on arrival (selection bypass from prior P10.5 still active)
- Click satellite A, then satellite B before A's fly completes → A's fly cancels cleanly, B's fly proceeds, no orphaned temp entities
- Predict failure → falls back to current position, logs warning, fly still happens
- Aircraft/vessel/fire click-to-fly behavior unchanged

Do not touch other entity types' flyTo targeting, theaters, trails, LOD thresholds (100km gate stays), selection bypass logic, or EntityPanel content.
