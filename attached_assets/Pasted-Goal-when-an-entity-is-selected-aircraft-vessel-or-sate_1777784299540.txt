Goal: when an entity is selected (aircraft, vessel, or satellite), draw a fading trail behind it showing recent path. Trail clears on deselect. Use Cesium's native PathGraphics.

Critical context: the previous selection-clobber bug was fixed in P7 — preserve that fix. The path-cleanup logic must check the PREVIOUS selectedEntity before the new one and clean only the old entity's path, not all entities' paths.

Implementation:

1. Aircraft trails (src/layers/AircraftLayer.ts):
   - Subscribe to store.selectedEntity changes via Zustand subscribe
   - On change, if previousSelected was an aircraft from THIS layer's Map, set that entity.path = undefined
   - If newSelected is an aircraft from THIS layer's Map, set entity.path = new Cesium.PathGraphics({
       leadTime: 0,
       trailTime: 600, // 10 minutes back
       width: 2.5,
       resolution: 2.0,
       material: new Cesium.PolylineGlowMaterialProperty({
         glowPower: 0.25,
         color: Cesium.Color.fromCssColorString('#22d3ee').withAlpha(0.85),
         taperPower: 0.5
       })
     })
   - The SampledPositionProperty already has historical samples from past polls — the trail renders those automatically
   - Only the SELECTED aircraft has a path — never trail every aircraft

2. Vessel trails (src/layers/VesselLayer.ts):
   - Same pattern
   - trailTime: 1800 (30 minutes — ships are slower)
   - color: '#34d399' (emerald, matches vessel color)
   - width: 2

3. Satellite trails (src/layers/SatelliteLayer.ts) — different approach:
   - Satellites use PointPrimitiveCollection, not Cesium.Entity
   - Solution: when a satellite is selected, create a TEMPORARY Cesium.Entity for that satellite specifically
   - Populate its position with a SampledPositionProperty by querying the SGP4 worker for positions at 30-second intervals across [now - 30min, now + 30min]
   - This temp entity's only purpose is the path — set its show=false or no point/billboard graphics, just the path
   - PathGraphics: leadTime: 1800, trailTime: 1800 (30 min each way — past orbit + predicted future as one continuous arc)
   - Material: PolylineGlowMaterialProperty with color '#a855f7' (purple) at 0.7 alpha, width 1.5
   - On deselect, destroy the temp entity completely

4. Restricted airspace, jamming hexes do NOT get trails — they're regional overlays, not moving entities. If selectedEntity.type === 'airspace', neither layer adds a path.

5. Use a single Zustand subscription per layer that fires on selectedEntity changes:
   - Store the previous value via useRef or closure
   - On change: clean up old entity's path if it belongs to this layer; add path to new entity if it belongs to this layer
   - On null: just clean up old

Definition of done:
- Click a plane → cyan glowing trail appears behind it, ~10 minutes of past flight path. Trail tapers from full opacity at the plane to transparent at the tail.
- Click a different plane → previous plane's trail disappears, new plane's trail appears.
- Click empty space → trail disappears.
- Click a vessel → emerald wake trail behind it (30 min).
- Click a satellite → purple orbital arc visible, 30 min past + 30 min future as continuous path.
- Performance stable even with rapid select/deselect.
- No regressions: aircraft layer empty-click bugfix from P7 still works (other selections survive aircraft empty clicks).

Do not add new layers, do not modify the theater system, do not touch anything outside the trail rendering logic.
