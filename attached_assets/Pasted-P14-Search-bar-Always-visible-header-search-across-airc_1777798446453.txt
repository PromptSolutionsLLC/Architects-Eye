P14: Search bar. Always-visible header search across aircraft, vessels, satellites, quakes, cables. Click a result → opens entity card + click-to-fly via P10.5.

LOCATION + STYLING:

1. components/HeaderBar.tsx — add SearchBox component centered horizontally between the existing left-side title block ("ARCHITECT'S EYE" + UNCLASSIFIED//OSINT pill) and the right-side LIVE pill / UTC clock.

2. SearchBox dimensions: 320px wide, 32px tall, monospace font matching existing UI font, dark background matching panel style, 1px border (neutral when unfocused, amber #fbbf24 when focused).

3. Placeholder text: "SEARCH ENTITIES..." in dim grey (#525252 or similar), monospace, uppercase preserved.

4. Input visible on every load. No keyboard shortcut required. (`/` key shortcut OUT of scope for tonight.)

QUERY BEHAVIOR (components/SearchBox.tsx — new file):

5. Local state: query: string, results: SearchResult[], focused: boolean.

6. Debounce query input by 150ms before recomputing results (prevents query storm during fast typing).

7. SearchResult shape: { type: 'aircraft' | 'vessel' | 'satellite' | 'quake' | 'cable', id: string, label: string, sublabel: string }
   - aircraft: label = callsign || icao24, sublabel = "AIRCRAFT · " + icao24
   - vessel: label = vesselName (from ShipStaticData) || mmsi, sublabel = "VESSEL · MMSI " + mmsi
   - satellite: label = satellite name (from TLE line0), sublabel = "SATELLITE · NORAD " + noradId
   - quake: label = "M" + magnitude.toFixed(1), sublabel = "QUAKE · " + place
   - cable: label = cable.name, sublabel = "CABLE · " + slug

8. Query algorithm:
   - Empty query → results = [], dropdown hidden.
   - query.length >= 1 → search aircraft, vessels, quakes, cables.
   - query.length >= 3 → ALSO search satellites (gated to avoid 15k-record scan on every keystroke).
   - Match: case-insensitive substring match on label OR sublabel-relevant fields. For aircraft: match callsign OR icao24. For vessels: match name OR mmsi. For satellites: match name OR norad string. For quakes: match place string OR "M" + magnitude shorthand. For cables: match name OR slug.
   - Iterate the corresponding store maps directly. No prebuilt index — overkill for tonight.
   - Sort within each type by best-match-first (exact prefix match > prefix match > substring). Then concat across types in this priority order: satellite > aircraft > vessel > cable > quake.
   - Cap total results at 10. If more matches exist, show "+N more" footer line in the dropdown.

9. Performance note for satellite scan: with 15,304 satellites and a substring match per keystroke (after debounce + length>=3 gate), worst case is ~15k string checks per query. Acceptable on modern JS. If FPS drops during typing, log [SEARCH PERF] keystroke_ms=X — only optimize if it's actually slow. Do NOT prebuild a search index tonight.

DROPDOWN UI:

10. Dropdown appears below the search box when focused === true AND results.length > 0.
    - Width: matches search box (320px).
    - Background: same dark panel style.
    - 1px top border continuous with search box border.
    - Each result row: 36px tall, 8px horizontal padding, hover state with faint highlight, click target full row.
    - Row content: icon dot color-matched to entity type (aircraft cyan, vessel emerald, satellite purple, quake yellow/orange/red, cable teal — match LayerToggles dots), then label in monospace bold, then sublabel in dim monospace small.
    - Keyboard navigation: ArrowUp/ArrowDown to highlight rows, Enter to select highlighted, Escape to clear query and blur.
    - Click outside the search area → blur, hide dropdown.

ACTION ON SELECT:

11. Selecting a result (Enter or click):
    - Look up the full entity from its store map by id.
    - If entity not found (was removed between search and click) → log [SEARCH STALE] type=X id=Y, show inline error toast "Entity no longer available" for 2s, do nothing else.
    - If entity found → call the SAME pipeline P11 central click handler uses: store.replaceUnpinnedCards({type, id, data}) followed by flyToInspect({type, id, data}).
    - This reuses P10.5 click-to-fly exactly (predict for satellites, range/pitch per type, theater coordination via isTheaterFlying, rapid-cancel on subsequent searches/clicks).
    - Clear the search query, blur the input.

12. If isTheaterFlying === true at selection time → card opens but flyTo is suppressed (existing P10.5 behavior). Do not special-case.

LAYER VISIBILITY GUARANTEE:

13. If user searches for a type whose layer is currently toggled OFF (e.g. searches "ISS" while satellites toggle is off), force-enable that layer's visibility on selection: store.setLayerVisibility(type, true). Otherwise the user clicks a result and the entity is invisible.
    - Apply for: aircraft, vessels, satellites, quakes, cables.
    - Do NOT auto-enable jamming, airspace, fires (these aren't searchable anyway, but explicit allowlist).
