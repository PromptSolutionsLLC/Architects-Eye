Goal: ~22,000 catalogued satellites visible on the globe, propagated client-side at 60fps via SGP4 in a Web Worker.

Data source: Celestrak. Fetch ONCE on app load:
https://celestrak.org/NORAD/elements/gp.php?GROUP=active&FORMAT=tle
Returns 3-line element sets, ~22k satellites total.

The Celestrak fetch will need to go through the existing Express proxy (CORS) — add a /api/tle route that fetches the URL above and returns the raw text body. Cache it server-side for 24 hours since TLEs only update daily.

Implementation:

1. server/src/routes/tle.ts — new Express route GET /api/tle:
   - Fetches the Celestrak TLE URL
   - Caches result in-memory for 24 hours
   - Returns the raw text response with appropriate Content-Type
   - On upstream error, return cached data if available, else 502

2. src/utils/tle.ts — parse 3LE format into array of { name, line1, line2 }

3. src/workers/sgp4.worker.ts — Web Worker that:
   - On 'init' message receives the array of TLE entries
   - Precomputes satrec for each via satellite.twoline2satrec
   - On 'tick' message receives a Date (the current Cesium clock time), propagates ALL satellites:
     - satellite.propagate(satrec, date) → eciToGeodetic with satellite.gstime
   - Posts back a Float32Array packed [index, longitude_rad, latitude_rad, altitude_km] for each successfully propagated sat
   - Skips satellites where propagation fails (decayed, etc.)

4. src/layers/SatelliteLayer.ts:
   - On mount: fetch /api/tle, parse, init worker
   - Create ONE Cesium.PointPrimitiveCollection (NOT Entity API — primitives are 50× faster for this many points)
   - Drive ticks from Cesium.Clock onTick: send current time to worker, on response update primitive positions in batch
   - Color by category (parse from satellite name):
     * "STARLINK" → gray (#888888), pixelSize 2
     * "GPS BIIF" / "NAVSTAR" / "GPS BIII" → blue (#3b82f6), pixelSize 3
     * "GALILEO" / "BEIDOU" / "GLONASS" → light blue (#60a5fa), pixelSize 3
     * "ISS (ZARYA)" → yellow (#fbbf24), pixelSize 6
     * "DEB" or "R/B" anywhere in name → dark red (#7f1d1d), pixelSize 1.5
     * default → green (#10b981), pixelSize 2
   - Click handler on the PointPrimitiveCollection: on LEFT_CLICK, identify the picked primitive by index → look up the satellite name and TLE → store.setSelectedEntity({ type: 'satellite', id, data: { name, noradId, category, altitude, period } })
   - Compute orbital period from line2 mean motion: 1440 / meanMotion minutes

5. EntityPanel.tsx — extend to handle type 'satellite' and show: name, NORAD ID (parse from line1), category, current altitude in km, orbital period in minutes

6. Wire LayerToggles satellite toggle to layerVisibility.satellites — when off, hide the entire PointPrimitiveCollection. Show the count in the toggle row.

7. Performance budget: must stay above 30fps with all sats visible. If profiling shows worker tick > 50ms, throttle worker ticks to 1Hz instead of every Cesium clock tick. Use requestAnimationFrame coalescing.

Definition of done: zoom out to space view → see the satellite cloud spread over Earth. Find ISS (largest yellow point — it's at ~415km altitude in equatorial-ish inclination). Click a Starlink → its name shows correctly in the panel. Frame rate stays usable.

Do not add vessels or jamming layers. Satellites only. Do not break the existing aircraft layer.
