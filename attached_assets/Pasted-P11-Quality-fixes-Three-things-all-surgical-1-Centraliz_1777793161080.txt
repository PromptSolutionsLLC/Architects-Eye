P11: Quality fixes. Three things, all surgical.

1. Centralized LEFT_CLICK pick handler

Currently each layer (Aircraft, Vessel, Satellite, Airspace, Fire) attaches its own LEFT_CLICK ScreenSpaceEventHandler. AircraftLayer's handler calls setSelectedEntity(null) on a miss, which clobbers selections from other layers when the aircraft handler runs after them.

Refactor to ONE central handler at the Viewer level:

- viewer.scene.pick(movement.position) called once per click
- A pick-resolver registry: each layer exposes a resolver function(pickedObject) → SelectedEntity | null at mount time (Viewer.tsx wires the registry)
- Iterate resolvers in priority order: airspace → fire → satellite → aircraft → vessel. First non-null wins.
- If all resolvers return null → setSelectedEntity(null)
- After setSelectedEntity, dispatch flyToInspect(...) using the resolved type per existing P10.5 click-to-fly logic. Theater coordination unchanged (isTheaterFlying still gates fly).
- Each layer's existing per-layer LEFT_CLICK handler is REMOVED. Layers register their resolver only.

Resolver contract per layer:
- AircraftLayer: input picked object → if picked.id is an aircraft Entity, return {type:'aircraft', id: icao24, data: aircraftStore.get(icao24)}; else null
- VesselLayer: same pattern, mmsi
- SatelliteLayer: handle BOTH PointPrimitive (picked.primitive instanceof PointPrimitive with picked.primitive.id matching satellite registry) AND glTF Entity (picked.id instanceof Entity, properties bag has type='satellite')
- AirspaceLayer: GroundPrimitive resolution; if picked is one of the 5 zones, return {type:'airspace', id: zoneName, ...}
- FiresLayer: PointPrimitive with id = fireIndex

Verify after change:
- Click each entity type → correct EntityPanel, no clobber, click-to-fly fires
- Click empty space → selection clears
- During theater fly, click any entity → EntityPanel opens, theater not interrupted (existing P10.5 guarantee)

2. Vessel teleport detector

In VesselLayer's AISStream position update ingest path:
- Compute distance between previous stored position and incoming position using Cesium.Cartesian3.distance OR haversine on lat/lon. Convert to nautical miles.
- Compute time delta in seconds between previous lastUpdate timestamp and incoming timestamp.
- If distance > 100 nm AND time delta < 60 seconds → drop the update entirely (no store mutation, no entity position change).
- Log dropped updates: console.warn('[VESSEL TELEPORT DROP]', {mmsi, prev_lat, prev_lon, new_lat, new_lon, delta_nm: X, delta_s: Y}).
- Do NOT drop on time delta >= 60s (legitimate slow update after vessel was out of range). Do NOT drop on distance <= 100nm regardless of time.
- First-ever update for an MMSI (no prior position) is never dropped.

3. Jamming hover tooltip

JammingLayer hexes currently have no hover affordance. Add Cesium ScreenSpaceEventType.MOUSE_MOVE handler at the Viewer level (NOT a per-layer handler — same architectural reason as the click consolidation):

- On mouse move, viewer.scene.pick(endPosition).
- If picked is a jamming hex Primitive (identify via picked.primitive type or stamped property), compute or look up the hex's intensity value (the same value driving the red→orange gradient) and h3 index.
- Render tooltip: positioned absolute div anchored top-right of cursor with 12px offset, monospace font (existing UI font), dark background (#0a0a0a or matches existing panel bg), 1px amber border (#fbbf24 or matches header amber), padding 6px 10px, pointer-events:none, z-index above globe canvas.
- Tooltip content (three lines):
  H3 INDEX: <hex_id>
  INTENSITY: <value rounded to 2 decimals>
  SOURCE: GPSJam.org · 2026-05-01
- On mouseout of any hex (picked is null OR not a jamming hex), hide tooltip.
- Tooltip ONLY appears when JammingLayer is toggled visible. If layer is off, no tooltip even if hexes were briefly rendered during HMR.

Definition of done:
- Click satellite/aircraft/vessel/fire/airspace → correct EntityPanel populates via single central handler, click-to-fly fires for non-airspace types, no flicker or clobber
- Click empty ocean → selection clears, no fly
- During theater fly, click anything → panel opens, theater preserved
- Vessel teleport scenario: MMSI jumps from Atlantic to Pacific in 30s → console shows [VESSEL TELEPORT DROP], entity stays at last good position, AIS continues processing legitimate updates
- Toggle Jamming on → hover red hex → tooltip appears with H3 + intensity + source; mouse off → tooltip hides; toggle Jamming off → tooltip never appears

Do not touch P9 fires fetch logic, theaters, trails, tileset settings, P10 LOD or selection bypass, P10.5 click-to-fly targeting (only the dispatch wiring changes).
